  **## 0. КРИТИЧЕСКАЯ ДИРЕКТИВА (ОБЯЗАТЕЛЬНО ПРОЧИТАТЬ ПЕРВЫМ)**
  **### СТОП: Это НЕ лендинг**
  Если ты (Composer 2) делал до этого **одностраничный лендинг** — **немедленно остановись и переключись** на создание **многостраничного сайта**.
  **Почему это критично:**
  - Лендинг = 1 URL = можно ранжироваться максимум по 3-5 SEO-запросам
  - Цель этого проекта = ранжироваться **по 200+ запросам** = нужно **100+ отдельных страниц**
  - SEO-органика требует уникального URL на каждую тематическую группу запросов
  - Поисковики (Яндекс, Google) не ранжируют одну страницу по множеству несвязанных интентов
  **Что делать если уже есть лендинг:**
  - НЕ удаляй существующий код
  - Преврати текущий лендинг в **главную страницу** `/`) многостраничного сайта
  - Создай **иерархическую структуру URL** согласно разделу 5 этого документа
  - Добавь **навигацию и футер** со ссылками на новые разделы
  **### СТРОГИЙ ФОКУС: Только склады, только аутсорсинг**
  **Этот сайт посвящён ИСКЛЮЧИТЕЛЬНО:**

  **## 3. КРИТИЧЕСКИЕ SEO-ТРЕБОВАНИЯ**
  **### 3.1. Core Web Vitals (Lighthouse-метрики)**
  Каждая страница должна достигать:
  - **LCP (Largest Contentful Paint)** < 2.5 сек
  - **FID (First Input Delay)** < 100 мс
  - **CLS (Cumulative Layout Shift)** < 0.1
  - **Lighthouse Performance Score** > 90 (десктоп), > 80 (мобильный)
  **### 3.2. Рендеринг**
  - **ВСЕ страницы** рендерятся в SSR или SSG (не CSR). Проверка: выключить JS в браузере — контент должен быть виден.
  - **Dynamic routes** (например, `/personal/[profession]`) используют `generateStaticParams()` для SSG
  - Блог-статьи — **SSG при билде**
  - Страницы с CMS — **ISR (Incremental Static Regeneration)** с revalidate 3600 сек
  **### 3.3. Meta-теги (каждая страница)**
  **Обязательный минимум:**
  ```html
  <title>Уникальный заголовок страницы | Название бренда</title>
  <meta name="description" content="Уникальное описание 150-160 символов">
  <meta name="robots" content="index, follow">
  <link rel="canonical" href="https://домен.ru/текущий-url">
  *<!-- Open Graph (для соцсетей) -->*
  <meta property="og:title" content="...">
  <meta property="og:description" content="...">
  <meta property="og:image" content="...">
  <meta property="og:url" content="...">
  <meta property="og:type" content="website">
  *<!-- Twitter Card -->*
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:title" content="...">
  <meta name="twitter:description" content="...">
  *<!-- Hreflang (на будущее, если будет SPb/Regional версия) -->*
  <link rel="alternate" hreflang="ru" href="...">
  **Правила:**
  - `title` уникальный на каждой странице, 50-60 символов, содержит основной ключ
  - `description` уникальный, 150-160 символов, не дублирует title, содержит CTA
  - `canonical` обязателен на каждой странице
  - `og:image` минимум 1200x630px, оптимизированный WebP/JPG
  ### **3.4. Schema.org разметка (JSON-LD)**
  Реализовать микроразметку на всех страницах:
  **На главной:**
  {
  "@context": "https://schema.org",

  "provider": {...},
  "areaServed": [...],

  }
  **На статьях блога:**

  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "...",
  "author": {...},
  "datePublished": "...",

  **На FAQ-блоках:**

  **На хлебных крошках:**
  {
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [...]
  }
  ### **3.5. Sitemap и robots.txt**
  Генерировать автоматически при билде:
  `/sitemap.xml` (через `next-sitemap` или встроенный Next.js):
  - Включает **все** публичные URL
  - Приоритет: главная = 1.0, услуги = 0.9, профессии/отрасли = 0.8, блог-статьи = 0.7, кейсы = 0.6
  - Частота обновления: главная = daily, блог = weekly, остальные = monthly
  - В Yandex.Webmaster загрузить sitemap
  `/robots.txt`**:**
  User-agent: *
  Allow: /
  Disallow: /admin
  Disallow: /api
  Disallow: /_next
  Disallow: /*.json$
  User-agent: Yandex
  Allow: /
  Clean-param: utm_source&utm_medium&utm_campaign
  Sitemap: https://домен.ru/sitemap.xml
  Host: домен.ru
  ### **3.6. Правила URL**
  - **Все в нижнем регистре:** `/personal/gruzchiki` ✅, `/Personal/Gruzchiki` ❌
  - **Транслитерация:** кириллица → латиница в URL (`gruzchiki`, `komplektovshchiki`)
  - **Дефисы**, не подчёркивания: `vodsitel-pogruzchika` ✅, `vodsitel_pogruzchika` ❌
  - **Без лишних слов:** `/gruzchiki-moskva` ✅, `/uslugi-gruzchiki-v-gorode-moskva` ❌
  - **Иерархия:** `/personal/gruzchiki`, `/lokatsii/khimki` — отражает структуру
  - **Никаких .html, .php на конце**
  - **Trailing slash:** либо везде с ним (`/url/`), либо везде без (`/url`) — решение зафиксировать и соблюдать
  ### **3.7. Изображения**
  - **Формат:** WebP primary, AVIF fallback, JPG для старых браузеров
  - **Компрессия:** все изображения оптимизированы до < 200 KB (чаще 50-100 KB)
  - **Lazy loading:** все изображения below the fold с `loading="lazy"`
  - **Размер:** `width` и `height` обязательны в HTML (предотвращает CLS)
  - **Alt-теги:** обязательны, осмысленные, с ключом если уместно
  - **Next.js Image Component** (`next/image`) — использовать везде
  ### **3.8. Внутренняя перелинковка**
  - Каждая страница услуги ссылается на **5-10 связанных страниц**
  - В статьях блога — **3-5 ссылок на коммерческие страницы**
  - **Хлебные крошки** (breadcrumbs) обязательны на всех страницах кроме главной
  - **Связанные материалы** в футере статей
  - **Релевантные профессии** на страницах отраслей и наоборот
  ### **3.9. Запрещённые SEO-практики**
  - ❌ Скрытый текст (`color: white` на белом фоне, `display: none` для SEO)
  - ❌ Keyword stuffing (ключ в каждом предложении)
  - ❌ Дублирование контента между страницами
  - ❌ Автогенерация контента без вычитки
  - ❌ Клоакинг (показ разного контента поисковику и пользователю)
  - ❌ Покупные низкокачественные ссылки
  - ❌ Фейковые отзывы (Яндекс штрафует)

  ## **8. КАЧЕСТВО КОНТЕНТА (ПРАВИЛА)**
  ### **8.1. Тон и стиль**
  - **B2B экспертный**, но не сухой
  - **Конкретные цифры** вместо «много», «быстро», «эффективно»
  - **Короткие абзацы** (2-4 предложения)
  - **Подзаголовки** каждые 250-350 слов (поддержка читаемости)
  - **Списки и таблицы** (людей и Google любят структуру)
  - **Без воды** и канцелярита
  ### **8.2. Запрещённые фразы**
  - ❌ «Качественный аутсорсинг» (что значит «качественный»?)
  - ❌ «Профессиональный подход»
  - ❌ «Индивидуальный подход к каждому клиенту»
  - ❌ «Широкий спектр услуг»
  - ❌ «Большой опыт» (напиши сколько лет)
  ### **8.3. Обязательные элементы на каждой странице**
  - **Цифры:** сколько лет, сколько клиентов, сколько персонала
  - **Конкретные кейсы** (хотя бы 1-2 на каждой коммерческой странице)
  - **Чёткие тарифы** (от X ₽/час) или «по запросу с расчётом в 1 день»
  - **FAQ** 6-8 вопросов внизу
  - **CTA** явный и понятный
  - **Телефон + быстрый контакт** (WhatsApp/Telegram)
  ### **8.4. SEO-правила**
  - **Основной ключ** в: `title`, `H1`, первом абзаце, заголовке минимум 2 подразделов
  - **LSI-ключи** (семантически связанные) — 5-10 раз по тексту
  - **Плотность основного ключа:** 1-2% (не больше)
  - **Anchor-тексты** внутренних ссылок — осмысленные (не «тут»)
  ```
